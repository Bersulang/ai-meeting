# 后端架构说明

## 总体设计

仓库以 `admin` 模块为唯一后端应用模块，围绕以下目标组织：

- 面向展示的工程化结构
- 面向 AI 场景的多能力聚合
- 面向联调的统一接口风格

## 分层结构

### Controller 层

- 负责 HTTP 入参接收
- 对外暴露 REST / SSE / 文件上传 / WebSocket 辅助接口
- 使用统一返回模型 `Result<T>`

### Service 层

- 承担业务编排和第三方调用
- 包含 AI 对话、面试流程、音频处理、TTS、题库采集等能力
- 通过缓存、线程池和第三方 SDK 协调复杂流程

### DAO / Repository 层

- MyBatis Plus / Spring Data 双栈并存
- 分别处理结构化数据、文档数据和缓存读取

### Common / Config 层

- 全局异常处理
- 鉴权与用户上下文
- 线程池、SSE、Redis、Mongo、数据库等配置
- 运行时目录初始化与健康检查配置

## 运行时依赖

- MySQL：主业务数据
- Redis：会话缓存、限流、状态同步
- MongoDB：文档型数据与扩展存储
- 外部 AI 能力：OpenAI、讯飞、星辰工作流等

## 本轮工程化收尾重点

- 修正 Maven 打包入口
- 补齐健康检查与 Docker 部署链路
- 将临时文件、日志目录移出源码树
- 收敛调试输出和接口校验边界
- 为仓库补齐开源协作文件与模板

## 后续仍可继续加强的方向

- 数据库迁移工具（Flyway / Liquibase）
- 更完整的接口契约文档
- 指标、链路追踪与可观测性
- 更细粒度的集成测试与容器测试
